Final Project

Jessica Douma | HCDE 533 | Autumn 2022



Digitally Fabricated Moss Poles and Accessories

Day Progress
11/22 Parametrically designed the anchor base of the moss pole w/o the mounting options. Waiting until I've designed and tested the screw mechanism before incorporating the mounting holes.

Tried using this tutorial to create a screw mechanism. Played around with the different inputs and created an STL for a quick test print. This option desperately failed since the bottom/top of the "screw" was square and didn't allow for the screw to naturally screw into the cap I created.

Next, I tried some of the Grasshopper scripts from this forum. I chose some random inputs and baked both the male and female screws. The female screw was an open polysurface and I couldn't figure out how to manipulate it into a cap like shape. So instead I used the male shape to BooleanSplit it from a cylinder to create a cap and ran another test print. This was slightly more succesful than my first test print, but the cap didn't have any tolerance for the screw, which meant it only went in about a half turn before stopping.

To simplify my parametric modelling for each separate component, I decided to model each major piece in their own file first, and then I'll merge them together as needed. This made it easier to organize and incrementalley build on my work.
11/23 Followed this YouTube Tutorial to parametrically design my own threads along a cylinder. It was incredibly straightforward and easy to follow. I adjusted the inputs to be similar to the size I expect one of the screw connections to be and kicked off a test print.
11/24 Followed this YouTube Tutorial to parametrically design my own caps for the threads I created the previous day. This video walked through created the cap with a tolerance (the part that my previous test prints missed) so I was hopeful this would work. I followed the tutorial and ran a test print referencing the same inputs as the previous day to see how the two pieces would work together.
11/25 With both the screw and the threaded nut printed, I tested them out together and - THEY WORKED. Now that I knew I the screw mechanism was taken care of, I turned to the next most complex (at least for me since I haven't done this before) part which is designing out the shell for the actual moss pole piece. I followed this Parametric House Tutorial. The tutorial and resulting geometry relied on the LunchBox plugin, so I had to jump through some hoops to get it installed since they only offer an installer for Windows on their site. I followed the first part of the tutorial to build the pole mesh and then added a couple of my own details and kicked off a test print.

While I waited for the test print to finish, I made my grasshopper script for the plant clips. This proved a bit trickier than I thought since a lot of my curves perfectly intersected which made trimming with regions a no-go. I tried making them overlap slightly but this effected the geometry. I decided to create one script that would allow me to parametrically create the geometry that I needed in Grasshopper that I could bake and remove the extraneous inner edges in Rhino. I then created a second script that takes the resulting curve I tidied up in Rhino and creates an array of the curve for laser cutting.
11/26 I checked on my print in the morning and it looked really good! However, the settings I chose, the moss pole mesh cross sections were too thin, when I tried to remove the support they kept breaking. I updated the inputs in my parametric model to kick off another test print with thicker cross sections. And this is why I love parametric design - it's incredibly easy to iterate without duplicating effort.

When I came back to my test print, it looked great and I think the cross-section radius was looking better than the previous iteration, however this time around I printed with "everywhere" support rather than "tree" support (since it was difficult to remove the tree support from the last print) and it was impossible to remove. I kicked off another test print, this time with no support to see how it would fair. While that was running I started working on the anchor adapter geometry.
11/27 My moss pole test print was finally successful. The trick was printing with no support and ensuring that there's enough U divisions so the base of the moss pole has something to connect to when printing at the top.

Today I began integrating all of my parts. I pretty much designed each individual component but now I have to integrate the screw and thread mechanism across all of my designs. I started with the anchor base and kicked off a test print. For the diameter of my test print, I chose the bottom diameter of the pot that I plan to demo my moss pole with.

Next, I moved onto the anchor extender since that would be using the same screw dimensions as the base, since the anchor adapter would be screwing into it. Originally I had planned to have all of my components in a single file. However, after adding the screw mechanism to the anchor file, I realized that the computations were starting to slow down my computer. So I decided to keep each component separate. This means that I'll have to be super careful to make sure all the integrating pieces use the same inputs across files.

With the anchor extender finished, I moved onto integrating the screw mechanism into the adapter and moss pole sections. I synced the inputs across components and kicked off a bunch of low quality test prints to see how they integrated.
